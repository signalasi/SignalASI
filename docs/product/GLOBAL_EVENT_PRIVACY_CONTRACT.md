# Global Event and Private Data Contract

SignalASI maintains one encrypted Personal ASI above topic conversations. This document defines
the production boundary between authorized semantic events, long-term cognition, backup, and
destructive reset.

## Event Publisher Contract

`GlobalEventPublisherContract` owns the canonical publisher for every
`GlobalConversationEventType`. It covers:

- messages and conversation lifecycle;
- attachments and generated artifacts;
- tasks, tool lifecycle, and tool results;
- cognition decisions and user feedback;
- memory and knowledge lifecycle;
- authorization and resource lifecycle.

The host adds `event_schema`, `publisher_id`, and `semantic_class` after receiving an event.
Producers cannot override these values. A unit gate fails when a new event type has no publisher,
has multiple publishers, or leaves a required semantic class uncovered.

Events carrying a denied authorization state, paused tracking, private-mode metadata, or excluded
global visibility are rejected before persistence. Control events such as deletion, retraction,
authorization revocation, and resource removal remain admissible so stale evidence can be removed.

## Private Conversation Boundary

Private conversation content, attachments, artifacts, tool results, titles, and topic hints never
enter the global event queue. Only bounded lifecycle tombstones may be retained. Pausing tracking
has the same content boundary.

Deleting or replacing a source event emits a causal retraction. World items, graph relations,
context journal entries, generated artifacts, and delayed derived results cannot survive or
recreate retracted evidence.

## Backup Boundary

Every backup contains a machine-readable privacy manifest generated by
`AgentPrivateDataInventory`. The backup container is password-encrypted before it is written.

Always-exported user-owned data includes identity, profile, long-term memory, knowledge, workflows,
safety settings, Personal ASI state, self-model, voice settings, and configured device state.
Contacts and conversation history remain explicit user choices.

The following live authority and execution state is never exported:

- permission grants;
- run-start idempotency receipts;
- active Run workspaces and event ledgers;
- pending connector responses and Link delivery queues;
- MCP credentials and executable Skill installations;
- on-device Linux workspaces, exports, downloads, and caches.

These items must be reauthorized, reinstalled, or recreated on the destination device.

## Reset Boundary

Reset clears known encrypted stores first, then sweeps every application database, shared
preference file, internal file, no-backup file, external app file, and cache directory. The storage
master key and Signal identity are destroyed. A new identity is generated only after erasure, so a
reset device cannot silently retain its former trust authority.

## Release Gates

The Android test suite verifies:

1. every event type has exactly one host-owned publisher;
2. required message, file, decision, task, tool, and feedback classes are covered;
3. publisher metadata cannot be spoofed;
4. denied and paused content is rejected while revocation events remain usable;
5. every private store has an export and erase decision;
6. optional history and contacts obey the backup selection;
7. permission grants, receipts, runtime files, and credentials stay local;
8. private-mode content and causal deletion remain covered by the world, graph, journal, and
   observation regression suites.
